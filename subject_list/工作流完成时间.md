# 工作流完成时间

## 题目描述

给定一个工作流 `task_flow`，其中每个任务都用一个二元组表示：

```python
[task_type, pre_idx]
```

- `task_type` 表示任务类型
- `pre_idx` 表示该任务的前置任务下标
- 当 `pre_idx = -1` 时，表示该任务没有前置依赖

任务按其在 `task_flow` 中的顺序编号为 `0 ~ n-1`。

不同类型的任务有固定的执行通道和耗时：

| 任务类型 | 执行通道 | 执行耗时 |
| --- | --- | --- |
| A | 1 | 10 |
| B | 2 | 12 |
| C | 3 | 17 |
| D | 4 | 9 |
| F | 5 | 5 |

工作流的执行规则如下：

1. 不同通道之间可以并行执行任务。
2. 同一通道同一时刻只能执行一个任务，因此同一通道上的任务必须串行执行。
3. 某个任务只有在它的前置任务完成后才能开始执行。
4. 一个任务的开始时间为：

```text
max(该任务所属通道的可用时间, 前置任务的结束时间)
```

5. 如果同一时刻同一通道上有多个已经满足执行条件的任务，则按照它们在 `task_flow` 中的下标从小到大依次执行。

请你计算：该工作流中所有任务全部执行完成所需的总时间。

## 输入描述

给定一个二维数组 `task_flow`：

- `task_flow[i][0]` 为任务类型，取值范围为 `A / B / C / D / F`
- `task_flow[i][1]` 为前置任务下标，`-1` 表示无前置任务

可以认为输入保证合法，即：

- 任务类型一定有效
- 前置任务下标不是 `-1` 时，一定指向某个合法任务
- 不存在循环依赖

## 输出描述

输出一个整数，表示该工作流全部完成的总耗时。

## 样例 1

**输入：**

```python
[['A', -1], ['A', -1], ['B', -1], ['C', 2]]
```

**输出：**

```python
29
```

**说明：**

- 任务 `0` 是 `A`，在通道 1 上执行，耗时 10，执行区间为 `[0, 10)`
- 任务 `1` 也是 `A`，同样在通道 1 上，但通道 1 需要等任务 `0` 完成后才能继续，因此执行区间为 `[10, 20)`
- 任务 `2` 是 `B`，在通道 2 上执行，无依赖，执行区间为 `[0, 12)`
- 任务 `3` 是 `C`，依赖任务 `2`，因此最早从时刻 12 开始，在通道 3 上执行，区间为 `[12, 29)`

最终总耗时为 `29`。

## 样例 2

**输入：**

```python
[['A', -1], ['B', -1], ['C', 0], ['D', 1], ['F', 3], ['A', 4]]
```

**输出：**

```python
36
```

**说明：**

- 任务 `0`：`A`，执行区间 `[0, 10)`
- 任务 `1`：`B`，执行区间 `[0, 12)`
- 任务 `2`：`C`，依赖任务 `0`，执行区间 `[10, 27)`
- 任务 `3`：`D`，依赖任务 `1`，执行区间 `[12, 21)`
- 任务 `4`：`F`，依赖任务 `3`，执行区间 `[21, 26)`
- 任务 `5`：`A`，依赖任务 `4`，且通道 1 也必须空闲，因此执行区间 `[26, 36)`

最终总耗时为 `36`。

## 备注

- 如果 `task_flow` 为空，则总耗时为 `0`。
- 每个任务最多只有一个前置任务。

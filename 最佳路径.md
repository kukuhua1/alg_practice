# 最佳路径

## 一、题目描述

给定系列单板间的有向连接关系 `connections`，其中：

```text
connections[i] = [srcBoardId, snkBoardId, weight]
```

表示从单板 `srcBoardId` 到单板 `snkBoardId` 有一条权值为 `weight` 的单向连接。

基于单板间的有向连接关系，从起始单板到目的单板可能有 `0`、`1` 或多条可达的有向路径。一条路径的权值是该路径上各连接的权值之和。

给定一系列格式为：

```text
[fromBoardId, toBoardId]
```

的查询命令 `querys`，请依次计算并返回其最佳路径权值；若某条最佳路径不存在，则该条最佳路径权值为 `-1`。

最佳路径规则如下：

1. 优先选择经过的连接个数最少的路径；
2. 若仍存在多条，则选择权值最小的路径。

## 二、输入说明

首个参数为 `connections`：

```text
0 < connections.length <= 500
```

第二个参数为 `querys`：

```text
0 < querys.length <= 500
```

编号与权值范围：

```text
0 <= srcBoardId, snkBoardId, fromBoardId, toBoardId < 1000
0 <= weight < 1000
```

## 三、输出说明

返回一个列表，其中每个值依次表示对应查询的最佳路径权值。

## 四、样例

### 样例 1

输入：

```plaintext
[[100, 101, 10], [102, 101, 5]]
[[100, 101], [102, 100]]
```

输出：

```plaintext
[10, -1]
```

解释：

```text
首次查询：单板 100 到单板 101，最少经过 1 次跳转可达，权值为 10。
再次查询：单板 102 到单板 100，无可达路径，权值为 -1。
```
